# Postgres + Node.js Example

This example adds a Node.js app that uses the PowerSync Node.js SDK to connect to the self-hosted stack. Auth uses a shared HS256 secret in a root `.env` file: PowerSync verifies JWTs, and the client app signs them (demo only; not recommended for production).

## Layout

```
local-postgres-node/
├── .env                 # PS_CLIENT_AUTH_KEY (shared secret; copy from .env.example)
├── powersync/           # PowerSync service config
│   ├── docker/
│   ├── link.yaml
│   ├── service.yaml     # client_auth with HS256 jwks
│   └── sync.yaml
├── src/                 # Node.js app
│   ├── auth.ts          # JWT signing with shared secret
│   ├── connector.ts     # PowerSync backend connector
│   ├── index.ts         # Main entry: connect, sync, query
│   └── schema.ts        # Client-side schema
├── package.json
└── README.md
```

## Steps to Create This Demo

### 1. Copy from local-postgres

Start from the [local-postgres](./local-postgres) example (or run `powersync init` and `powersync docker configure --database=postgres --storage=postgres`).

### 2. Add client_auth with HS256 to service.yaml

Update `powersync/service.yaml` to use JWT auth instead of static tokens:

```yaml
client_auth:
  audience: ['http://localhost:8080', 'http://127.0.0.1:8080']
  jwks:
    keys:
      - kty: oct
        alg: HS256
        kid: dev-key-1
        k: !env PS_CLIENT_AUTH_KEY

api: {}
```

### 3. Add shared secret to env files

Create a root `.env` (and ensure `powersync/docker/.env` includes it):

```bash
# Generate secret: openssl rand -base64 32 | tr '+/' '-_' | tr -d '='
PS_CLIENT_AUTH_KEY=<base64url-encoded-secret>
```

Update `powersync/docker/docker-compose.yaml` to load the root env:

```yaml
env_file:
  - .env
  - ../../.env
```

### 4. Initialize the Node.js app

```bash
pnpm init
pnpm add @powersync/node better-sqlite3 dotenv jose
```

Create the connector, schema, and auth module as in `src/`.

### 5. Start PowerSync and run the app

```bash
powersync docker start
pnpm build && pnpm start
```

## Usage

### 1. Copy env and install deps

```bash
cp .env.example .env
pnpm install
```

### 2. Start PowerSync

```bash
cd powersync && powersync docker start
cd ..
```

### 3. Run the app

```bash
pnpm build
pnpm start
```

The app connects to PowerSync, waits for the first sync, and prints synced todos.

### Optional env vars

- `POWERSYNC_URL` – default `http://localhost:8080`
- `USER_ID` – JWT subject (default `demo-user`)

---

## Security note

Using a shared HS256 secret for both PowerSync and the client is **not recommended for production**. In production, tokens should be generated by a secure backend; the client should call an API to obtain a JWT without ever seeing the signing key.
